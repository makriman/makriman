## Operating System

> Build beats decks. Ship the weird prototype. Put it in front of users. Watch what breaks. Make the system sharper.

I started with quiz tools because I wanted learning to feel less dead. That turned into websites, NLP experiments, AI tutors, RAG commerce, Shopify apps, agent tooling, and now control planes for AI-native software.

The pattern is basically the same every time: find a real workflow, build the smallest useful machine around it, and let users tell you where the abstraction is lying.
